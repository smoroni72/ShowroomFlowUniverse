# Asset Placement Rules

- I personaggi stanno solo in `assets/characters/`.
- Gli ambienti stanno solo in `assets/environments/`.
- I veicoli stanno solo in `assets/vehicles/`.
- Gli screenshot app stanno solo in `assets/props/app_screens/`.
- Gli episodi non devono contenere copie di asset condivisi, ma solo riferimenti in `source_assets.md`.
- Gli export finali stanno nell'episodio.
- Gli output Autopilot stanno in `outputs/autopilot/`.
- Gli esperimenti stanno in `archive/experiments/`.
- I file dubbi stanno in `archive/review_queue/`.
- Lo stato canonico di un asset vive nel `manifest.yaml`, non nel nome della cartella.
- I nomi file devono essere descrittivi, in `snake_case`, e includere contesto produttivo quando serve.
